# Observed Issues

Living notes for bugs and UX issues noticed mid-workflow, organized by product area.

Each file captures raw observations — not committed work. Add freely; prune as things get fixed or triaged.

## Index

- [Account](account.md) — profile, sharing, settings
- [Planner](planner.md) — the weekly planner view
- [Camp list](camp-list.md) — browse/search (currently `/explore`)
- [Camp detail](camp-detail.md) — individual camp page
- [Kids](kids.md) — kid profiles
- [Onboarding](onboarding.md) — signup → first planner entry
- [Schedule](schedule.md) — shared/public schedule view
- [Submit](submit.md) — operator-facing submit flow
- [Cross-cutting](cross-cutting.md) — issues that span multiple areas
- [General design/](general%20design/) — design-system issues not tied to one product area (buttons, spacing, shadows, etc.)

## How to use

- Drop a bullet under the right file. No formatting rules.
- Include **why** / **where** when it matters — context fades faster than the observation.
- Move an issue to a GitHub issue once it's ready to be worked on, then delete the bullet here.
