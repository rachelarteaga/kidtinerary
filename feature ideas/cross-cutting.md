# Cross-cutting

Ideas that span multiple product areas (navigation, auth, notifications, design system, etc.).

## Ideas

- **Optimize for mobile web.** No intention to build a native app — mobile web is the primary surface. Every product area should be designed and QA'd mobile-first.
- **Create a logo and brand moment.** Kidtinerary needs a real logo and a signature brand moment (splash/loading/first-run — TBD) so the product feels owned, not generic.
- **Predictive address search everywhere.** Any input that accepts an address (kid home address, camp location, organization address, etc.) should use autocomplete/predictive search — not free-text. One shared component, consistent UX.
